1. **Строгое следование промпту:** Выполняй ТОЛЬКО то, что явно просит пользователь. Запрещено проявлять инициативу, менять логику, дизайн или структуру соседних блоков, если об этом не было прямого указания.
2. **Никакого скрытого рефакторинга:** Не пытайся "улучшить", оптимизировать или переписать существующий код вокруг места изменения. Не форматируй файлы целиком.
3. **Безопасность сборки:** Перед сохранением любого кода дважды проверь синтаксис (незакрытые теги, пропущенные скобки, сломанные импорты). Твои изменения не должны ломать компиляцию проекта.
4. **Проверка после изменений:** После каждого выполненного промпта и внесения правок в код, предлагай команду для проверки сборки в терминале (например, `npm run build` или `npm run dev`), чтобы убедиться, что проект не сломался.
5. **Точечные изменения:** Изменяй ровно столько строк кода, сколько минимально необходимо для решения задачи.
6. **Сохранение стиля дизайна:** Если пользователь просит добавить новый визуальный элемент, всегда копируй и переиспользуй CSS-классы аналогичных элементов, которые уже есть в проекте.
